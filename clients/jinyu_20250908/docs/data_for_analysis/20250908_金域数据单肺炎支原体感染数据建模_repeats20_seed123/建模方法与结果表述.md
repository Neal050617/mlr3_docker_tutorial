# 肺炎支原体感染预测模型的建模方法与结果表述

## 一、材料与方法 - 建模部分

### 论文发表版本(完整段落形式)

本研究收集了单独肺炎支原体感染患者的临床数据,包括人口学特征、临床症状、实验室检查指标和影像学检查结果等。数据预处理过程中,对连续型变量采用历史插补法(historical imputation)处理缺失值,对分类型变量采用众数插补法(mode imputation),并移除常数特征。将数据按照9:1的比例随机划分为训练集和测试集,训练集用于模型构建和特征选择,测试集用于独立验证模型性能。为从高维临床数据中识别与肺炎支原体感染相关的关键预测因子,本研究采用基于MLR3框架的集成特征选择(Ensemble Feature Selection, EFS)方法。该方法集成了四种具有不同归纳偏置的机器学习算法作为基础学习器,包括XGBoost(基于梯度提升的决策树算法,擅长捕捉非线性关系和特征交互)、Random Forest(基于随机森林的集成学习算法,通过置换重要性评估特征贡献)、Support Vector Machine(基于线性核的支持向量机,适用于高维数据的线性判别)和LightGBM(基于梯度提升的轻量级算法,采用叶子优先生长策略)。采用递归特征消除(Recursive Feature Elimination, RFE)算法,设定目标特征数为20个,特征保留比例为85%,在每次迭代中训练模型并计算特征重要性,移除最不重要的15%特征,直至达到目标特征数。为评估特征选择的稳定性和模型性能,采用双层嵌套重采样策略:外层对训练集进行100次子采样(每次随机抽取80%的样本),内层在每个子采样数据集上进行5折交叉验证,该策略共执行500次模型训练,确保特征选择结果的统计可靠性。通过Jaccard系数量化不同数据子集上选择特征的一致性(系数>0.5表示稳定,>0.7表示高度稳定),通过帕累托前沿分析构建特征数量与模型性能的关系曲线,并采用膝点检测(knee point detection)自动确定最优特征数,同时应用One Standard Error Rule在保证性能的前提下选择特征数较少的简约模型以防止过拟合。采用SAV(Sum of Absolute Values)方法对四种算法的特征重要性进行综合排名,首先将各算法的重要性分数标准化至[0,1]区间,然后根据算法的交叉验证性能和稳定性进行加权求和,最终选择综合得分最高的20个特征作为最优特征子集。基于EFS选定的特征子集,构建五种机器学习预测模型,包括GLMNet(弹性网络正则化逻辑回归模型,结合L1和L2惩罚项)、XGBoost(极端梯度提升模型)、Random Forest(随机森林模型)、SVM(支持向量机模型)和LightGBM(轻量级梯度提升模型)。对每个模型采用随机搜索(random search)策略进行超参数优化,搜索次数设定为100次,在5折内层交叉验证框架下评估每组超参数的性能,选择交叉验证AUC最高的超参数配置作为最优参数。采用多层次评估策略全面评价模型性能,包括在训练集上进行5折交叉验证计算平均AUC、准确率和分类误差,使用全部训练数据训练最优模型评估拟合效果,以及在独立测试集上评估模型泛化能力。性能指标包括AUC(受试者工作特征曲线下面积,评估模型整体判别能力)、准确率(正确分类样本的比例)和分类误差(错误分类样本的比例)。对于支持特征重要性计算的模型(Random Forest, XGBoost, LightGBM, GLMNet),提取各模型的特征重要性得分,并通过可视化展示关键预测因子的相对贡献,同时对预测概率分布进行Wilcoxon秩和检验,评估模型区分阳性和阴性样本的统计显著性。所有分析均基于R语言(版本4.4.2)和MLR3机器学习框架(mlr3 v0.23.0, mlr3fselect v1.3.0, mlr3learners v0.10.0, mlr3tuning v1.3.0)实现。机器学习算法使用以下R包:XGBoost (v1.7.8.1)、ranger (v0.17.0)、e1071 (v1.7.16)、lightgbm (v4.6.0)和glmnet (v4.1.8)。采用双侧检验,P < 0.05认为差异具有统计学意义,模型性能的95%置信区间通过Bootstrap方法估计。

### 详细版本(分节形式,供参考)

### 1.1 数据预处理与划分

本研究收集了单独肺炎支原体感染患者的临床数据,包括人口学特征、临床症状、实验室检查指标和影像学检查结果等。数据预处理过程中,对连续型变量采用历史插补法(historical imputation)处理缺失值,对分类型变量采用众数插补法(mode imputation),并移除常数特征。将数据按照9:1的比例随机划分为训练集和测试集,训练集用于模型构建和特征选择,测试集用于独立验证模型性能。

### 1.2 集成特征选择方法

为了从高维临床数据中识别与肺炎支原体感染相关的关键预测因子,本研究采用基于MLR3框架的集成特征选择(Ensemble Feature Selection, EFS)方法。该方法结合了多种机器学习算法的优势,通过以下步骤实现稳健的特征选择:

#### 1.2.1 递归特征消除策略

采用递归特征消除(Recursive Feature Elimination, RFE)算法,设定目标特征数为20个,特征保留比例为85%。在每次迭代中,算法训练模型并计算特征重要性,移除最不重要的15%特征,直至达到目标特征数。通过交叉验证性能评估,选择最优特征子集。

#### 1.2.2 多算法集成策略

集成了四种具有不同归纳偏置的机器学习算法作为基础学习器:
- **XGBoost**: 基于梯度提升的决策树算法,擅长捕捉非线性关系和特征交互
- **Random Forest**: 基于随机森林的集成学习算法,通过置换重要性评估特征贡献
- **Support Vector Machine (SVM)**: 基于线性核的支持向量机,适用于高维数据的线性判别
- **LightGBM**: 基于梯度提升的轻量级算法,采用叶子优先生长策略

#### 1.2.3 双层重采样验证

为评估特征选择的稳定性和模型性能,采用双层嵌套重采样策略:
- **外层重采样**: 对训练集进行100次子采样,每次随机抽取80%的样本,用于评估特征选择在不同数据子集上的稳定性
- **内层重采样**: 在每个子采样数据集上进行5折交叉验证,用于无偏估计模型性能

该策略共执行500次模型训练(100次子采样 × 5折交叉验证),确保特征选择结果的统计可靠性。

#### 1.2.4 稳定性与性能评估

通过以下指标评估特征选择质量:
- **Jaccard系数**: 量化不同数据子集上选择特征的一致性,系数>0.5表示稳定,>0.7表示高度稳定
- **帕累托前沿分析**: 构建特征数量与模型性能的帕累托前沿曲线,通过膝点检测(knee point detection)自动确定最优特征数
- **One Standard Error Rule**: 在保证性能的前提下,选择特征数较少的简约模型,防止过拟合

#### 1.2.5 特征综合排名

采用SAV(Sum of Absolute Values)方法对四种算法的特征重要性进行综合排名。首先将各算法的重要性分数标准化至[0,1]区间,然后根据算法的交叉验证性能和稳定性进行加权求和,最终选择综合得分最高的20个特征作为最优特征子集。

### 1.3 预测模型构建

基于EFS选定的特征子集,构建五种机器学习预测模型:
- **GLMNet**: 弹性网络正则化逻辑回归模型,结合L1和L2惩罚项
- **XGBoost**: 极端梯度提升模型
- **Random Forest**: 随机森林模型
- **SVM**: 支持向量机模型
- **LightGBM**: 轻量级梯度提升模型

### 1.4 超参数优化

对每个模型采用随机搜索(random search)策略进行超参数优化,搜索次数设定为100次。在5折内层交叉验证框架下评估每组超参数的性能,选择交叉验证AUC最高的超参数配置作为最优参数。

### 1.5 模型性能评估

采用多层次评估策略全面评价模型性能:
- **交叉验证评估**: 在训练集上进行5折交叉验证,计算平均AUC、准确率和分类误差
- **训练集评估**: 使用全部训练数据训练最优模型,评估拟合效果
- **测试集评估**: 在独立测试集上评估模型泛化能力

性能指标包括:
- **AUC (Area Under the ROC Curve)**: 受试者工作特征曲线下面积,评估模型整体判别能力
- **准确率 (Accuracy)**: 正确分类样本的比例
- **分类误差 (Classification Error)**: 错误分类样本的比例

### 1.6 特征重要性分析

对于支持特征重要性计算的模型(Random Forest, XGBoost, LightGBM, GLMNet),提取各模型的特征重要性得分,并通过可视化展示关键预测因子的相对贡献。对预测概率分布进行Wilcoxon秩和检验,评估模型区分阳性和阴性样本的统计显著性。

### 1.7 统计分析

所有分析均基于R语言(版本4.4.2)和MLR3机器学习框架(mlr3 v0.23.0, mlr3fselect v1.3.0, mlr3learners v0.10.0, mlr3tuning v1.3.0)实现。机器学习算法使用以下R包:XGBoost (v1.7.8.1)、ranger (v0.17.0)、e1071 (v1.7.16)、lightgbm (v4.6.0)和glmnet (v4.1.8)。采用双侧检验,P < 0.05认为差异具有统计学意义。模型性能的95%置信区间通过Bootstrap方法估计。

## 二、结果 - 建模结果表述

### 论文发表版本(完整段落形式)

通过集成特征选择方法,从原始的多个临床和实验室指标中识别出20个与肺炎支原体感染密切相关的预测因子。在双层嵌套重采样验证框架下(100次子采样 × 5折交叉验证),四种基础学习器表现出较高的特征选择稳定性,其中Random Forest和XGBoost的Jaccard系数均达到0.6,表明所选特征在不同数据子集上具有良好的一致性。帕累托前沿分析显示,当特征数量达到约20个时,模型性能达到膝点,继续增加特征数量带来的性能提升边际效应显著递减(<2%),因此选择20个特征作为最优特征子集。基于SAV综合排名方法,识别出的关键预测因子按重要性降序包括:铁蛋白(急性期反应蛋白,炎症和感染的重要生物标志物)、免疫球蛋白IgE(反映机体免疫应答状态)、白蛋白水平(营养状态和肝功能指标)、DD二聚体(凝血功能指标,与血栓形成和炎症反应相关)、IL-8(促炎细胞因子,免疫反应的关键介质)、补体C3(补体系统成分,参与免疫防御)、TNF-α(肿瘤坏死因子,重要的促炎细胞因子)、中性粒细胞计数/淋巴细胞计数比值(反映机体炎症和免疫状态的综合指标)、补体C4(补体系统成分,参与免疫调节)和体温最高值(发热程度的直接反映)等,这些特征涵盖了炎症反应标志物、免疫功能指标、凝血功能参数和临床症状等多个维度,体现了肺炎支原体感染的多系统病理生理特征。基于选定的20个特征,构建了五种机器学习预测模型并进行性能评估。在5折交叉验证中,SVM模型表现最优(AUC = 0.794, 准确率70.0%),其次为GLMNet模型(AUC = 0.765, 准确率71.3%)、XGBoost模型(AUC = 0.753, 准确率65.0%)、Random Forest模型(AUC = 0.735, 准确率63.8%)和LightGBM模型(AUC = 0.732, 准确率66.3%)。在训练集上,Random Forest和LightGBM达到完美分类(AUC = 1.000, 准确率100%),XGBoost(AUC = 0.968, 准确率91.3%)、GLMNet(AUC = 0.955, 准确率88.8%)和SVM(AUC = 0.930, 准确率87.5%)也表现出优异的拟合效果。在独立测试集上,SVM模型表现最佳(AUC = 0.85, 准确率77.8%),显示出优异的泛化能力,而Random Forest、LightGBM和XGBoost的测试集AUC分别为0.60、0.60和0.55(准确率分别为77.8%、77.8%和66.7%),GLMNet的测试集AUC为0.50(准确率44.4%),提示部分模型存在一定程度的过拟合。特征重要性分析显示,不同模型对特征重要性的评估存在一定差异,但跨模型一致性较高的关键特征包括:免疫球蛋白IgE、白蛋白水平、中性粒细胞/淋巴细胞比值、铁蛋白和DD二聚体,这些指标在肺炎支原体感染的诊断和预后评估中具有重要临床价值。对各模型的预测概率分布进行Wilcoxon秩和检验,所有模型的预测概率在阳性和阴性组之间均存在统计学显著差异(P < 0.05),表明模型能够有效区分不同感染状态的患者。综合考虑交叉验证和测试集性能,SVM模型表现最为稳健,交叉验证AUC为0.794,测试集AUC为0.85,两者均为最高,且训练集(AUC = 0.930)与测试集性能差异较小,显示出良好的泛化能力和稳定性,为肺炎支原体感染的早期识别和风险分层提供了有效的预测工具。

### 2.1 集成特征选择结果

帕累托前沿分析显示,当特征数量达到约20个时,模型性能达到膝点,继续增加特征数量带来的性能提升边际效应显著递减(<2%),而模型复杂度大幅增加。因此,选择20个特征作为最优特征子集,在保证预测性能的同时实现模型简约性。

基于SAV综合排名方法,识别出的前10位关键预测因子包括(按重要性降序排列):

#### 2.3.2 训练集性能

- **Random Forest**: AUC = 1.000,准确率100%

#### 2.3.3 测试集性能

在独立测试集上,模型的泛化性能存在差异:

SVM模型在测试集上表现最佳(AUC = 0.85),显示出优异的泛化能力。Random Forest和LightGBM虽然训练集性能完美,但测试集AUC降至0.60,提示存在一定程度的过拟合。GLMNet模型在测试集上的AUC为0.50,接近随机分类水平,泛化能力较弱。

综合考虑交叉验证和测试集性能,**SVM模型**表现最为稳健,交叉验证AUC为0.794,训练集AUC为0.930,测试集AUC为0.85,三者均保持较高水平且差异较小,显示出良好的泛化能力和稳定性。**GLMNet模型**在交叉验证中表现优异(AUC = 0.765),训练集AUC为0.955,但测试集性能显著下降(AUC = 0.50),提示模型可能对训练数据过度拟合或测试集样本量较小导致的性能波动。**Random Forest和LightGBM**在训练集上达到完美分类(AUC = 1.000),但测试集AUC降至0.60,存在明显的过拟合现象。

### 2.5 特征重要性分析结果
