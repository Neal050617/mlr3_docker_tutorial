## 修正和升级摘要

| 问题 | 当前处理 | 新项目策略 |
|---|---|---|
| `模型比较` sheet 出现 `.x/.y` 或 `_thr` 列 | Step2 写 workbook 前已规范合并阈值指标 | 新 full rerun 不依赖 repair 脚本 |
| CV ROC 9 条曲线颜色不可区分 | Step2 使用稳定模型颜色映射 | 重跑 Step2 或从 RData 重画 |
| EFS 只有 2 个模型 | 低/中维 EFS 恢复 `xgb,rf,svm,lgbm` | 如果要改变特征选择过程，必须重跑 Step2 |
| 高维 SMOTE 分支过慢或不稳定 | 高维 SMOTE 跳过 ensemble EFS，走多方法组合 | 属于预期策略，不等于最终只训练少数模型 |
| SHAP 只能固定 train/test 样本 | 新增 case/control、显式样本、valid 和 force plot | 可基于已有 Step2 单独重跑 Step3 |
| 多个 SHAP 目录汇总混杂 | 汇总脚本支持 `--shap-dir-name` | 汇总时必须指定正式 SHAP 目录 |

## 已完成分析如何处理

如果旧项目只存在 workbook 展示问题，可以运行：

```bash
Rscript scripts/repair_mlr3_threshold_columns.R \
  --root=/path/to/result_root
```

这只修复展示，不改变模型训练、特征选择或 SHAP 结果。

如果项目需要恢复 4 learner EFS 口径，应重跑 Step2。只重画 EFS 诊断图不能改变 `selected_features`。

如果项目只需要升级 SHAP，可以基于已有 `efs.RData` 单独运行：

```bash
SHAP_WATERFALL_GROUPS=case,control \
SHAP_LOCAL_DISPLAY_FEATURES=10 \
SHAP_FORCE_PLOTS=TRUE \
bash scripts/run_mlr3_shap_all.sh /path/to/result_root
```

`SHAP_LOCAL_DISPLAY_FEATURES=10` 只限制 waterfall/force 单样本局部解释图；waterfall 保持多行 Top 10 局部贡献图，force 保持较低的箭头形状单行正负累积贡献条。importance、beeswarm、waterfall 和 force 只对较长特征名自动换行，默认超过 28 个字符时优先在靠近中点的 `-` 后断为最多两行；短特征名不换行。SHAP Excel 仍保留完整数值。
