- `Recall` 保留原指标名。
- `Sensitivity` 显式补写，数值等同于 `TP/(TP+FN)`，在本二分类任务中等同于 recall。
- `Specificity` 显式补写，数值为 `TN/(TN+FP)`。
- `混淆矩阵_Train/Test` 和每个单模型 sheet 都应能直接看到 `Sensitivity` 和 `Specificity`。

正式汇报至少展示：

- AUC
- AUPRC
- ACC
- MCC
- Sensitivity / Recall
- Specificity
- TP, FP, TN, FN
- Balanced_Score

`Balanced_Score` 只作为辅助展示列，不用于声明唯一最优模型。

9 种模型的基本解读注意事项如下：

| 方法 | 中文说明 | 解读注意 |
|---|---|---|
| `glmnet` | 正则化广义线性模型 | 可解释性较好，主要捕捉线性关系 |
| `xgboost` | 极端梯度提升树 | 小样本下需关注过拟合 |
| `rf` | 随机森林 | 稳定性较好，但单个规则不直观 |
| `svm` | 支持向量机 | 对尺度和参数敏感 |
| `naive_bayes` | 朴素贝叶斯 | 组学特征相关性强时需谨慎 |
| `kknn` | 加权 k 近邻 | 受距离度量和局部噪声影响 |
| `rpart` | 单棵决策树 | 解释直观但稳定性较弱 |
| `adaboost` | 自适应提升树 | 对噪声和异常样本敏感 |
| `nnet` | 单隐层神经网络 | 小样本下不稳定且解释性弱 |

SHAP 汇总：

```text
reproduce_gp01_hc_af_af_positive_fullrerun_20260526_1531/03_shap_output_summary/SHAP_OUTPUT_SUMMARY.md
reproduce_gp01_hc_af_af_positive_fullrerun_20260526_1531/03_shap_output_summary/shap_status_summary.tsv

reproduce_gp01_hc_af_af_positive_fullrerun_20260526_1531/03_shap_output_summary/shap_output_summary.xlsx
```

当前 AF-positive 结果的 SHAP 已完成并汇总。SHAP 用于解释模型，不用于定义最优模型。

Step3 支持为 train、test 和 valid 数据输出 SHAP；如果正式运行没有外部验证集，`valid` 会在 `step3_shap_status.tsv` 中记录为 skipped。waterfall 和 force 图默认按 `case,control` 各选 1 个样本；也可以通过 `SHAP_WATERFALL_SAMPLES` / `--waterfall_samples` 指定一个或多个样本 ID，支持 `train:ID`、`test:ID`、`valid:ID` 前缀。当前正式 SHAP PDF 中，importance、beeswarm、waterfall 和 force 只对较长特征名自动换行，默认超过 28 个字符时优先在靠近中点的 `-` 后断为最多两行；短特征名不换行；waterfall/force PDF 已按每个样本 `|SHAP|` 最大的 Top 10 特征重画；force 为较低的箭头形状单行正负累积贡献条，SHAP 值标在箭头内，Positive 特征名按箭头起点对齐，Negative 特征名按箭头尾部右对齐，完整 SHAP 数值仍以 `*_SHAP.xlsx` 为准。每个模型 Excel 中的 `*_waterfall_manifest` sheet 会记录实际命中的样本、分组和选择来源。

完成度可用 `*_SHAP.xlsx` 文件数复核：

```bash
find lina_20260515/reproduce_gp01_hc_af_af_positive_fullrerun_20260526_1531 \
  -path '*step3_shap_formal_case_control_force_20260526/*/*_SHAP.xlsx' | wc -l
```

每个分支理论最多 9 个模型 SHAP；当前 6 个分支共 54 个 SHAP Excel 输出，另有 216 个 waterfall PDF 和 216 个 force PDF。若后续重跑时模型 `AUC <= 0.5` 或实际运行失败，应以 `step3_shap_status.tsv` 记录，而不是手工假定完成。

SHAP 跳过规则中的 `AUC <= 0.5` 是“近似无分类能力模型不解释”的技术门槛，不是分类临界点，也不是模型选择标准。

当前 SHAP 是完整解释资源，而不是模型筛选依据。若用于论文主图，建议先根据完整指标表筛选表现稳定、混淆矩阵合理、MCC 较高的候选模型，再针对少数模型重新生成更高 `nsim` 的发表级 SHAP 图。
