# 01 建模设计与脚本来源

修订日期：2026-05-21

## 分析目标

在 `GP01.HC-AF` 中，以 `AF` 和 `HC` 为二分类结局，构建 5 个单组学诊断模型和 1 个多组学 biomarkers 联合诊断模型。

本轮正式结果以 `AF` 作为 case/positive class，`HC` 作为 control class。当前 v0.2.2 Step2 中二分类 positive class 来自 `gp[1]`，因此正式运行必须显式传入：

```bash
--gp AF-HC
```

不能在模型完成后再倒推 case/control 方向；若发现 completed run 的 positive class 反了，应将旧结果标记为方向错误并重跑，而不是仅修改表头或解释文字。旧方向结果已删除，不再作为正式结果入口。

| 模型 | 输入数据 | 任务 |
|---|---|---|
| `Model 0.1` | 非靶代谢 | AF vs HC |
| `Model 0.2` | 口腔细菌属 | AF vs HC |
| `Model 0.3` | 口腔真菌属 | AF vs HC |
| `Model 0.4` | 肠道细菌属 | AF vs HC |
| `Model 0.5` | 肠道真菌属 | AF vs HC |
| `Model 0.6` | 5 个单组学 Step2 入选 biomarkers 合并 | AF vs HC |

`Model 0.6` 不是直接拼接 5 套全量特征，而是从 `Model 0.1-0.5` 的 Step2 `selected_features` 中提取 biomarkers 后合并，再重新建模。

## 样本口径

原始分组文件：

```text
lina_20260515/map-group.txt
HC = 48
AF = 99
total = 147
```

5 套数据并非完全等样本：口腔真菌缺少 `HC20`，肠道真菌缺少 `HC37`。正式分析统一使用 145 个五组学共同样本：

```text
HC = 46
AF = 99
total = 145
排除样本: HC20, HC37
```

统一共同样本后，5 个单组学模型和联合模型可横向比较，也避免联合模型阶段引入额外缺失值处理。

正式运行前需要确认共同样本分组计数：

```text
AF = 99
HC = 46
```

## 原始数据核对

本项目原始数据来自 5 套组学矩阵和一个分组文件。正式建模前先核对样本方向、样本数量和特征数量。

| 数据 | 原始样本数 | 特征数 | 数据形态 | 备注 |
|---|---:|---:|---|---|
| 非靶代谢 | 147 | 59 | 样本为行，特征为列 | 与分组文件样本完整对应 |
| 口腔细菌属 | 147 | 144 | 特征为行，样本为列 | 属水平微生物丰度表 |
| 口腔真菌属 | 146 | 159 | 特征为行，样本为列 | 缺少 `HC20` |
| 肠道细菌属 | 147 | 210 | 特征为行，样本为列 | 属水平微生物丰度表 |
| 肠道真菌属 | 146 | 119 | 特征为行，样本为列 | 缺少 `HC37` |

正式分析没有对缺失组学样本做补齐，而是取 5 套组学共同样本。这样做牺牲了 2 个 HC 样本，但保证了 `Model 0.1-0.6` 使用同一批样本。

输入目录中还保留了以下辅助文件：

| 文件 | 用途 |
|---|---|
| `input_summary.tsv` | 每个模型输入矩阵的样本数、特征数和预处理 preset 汇总 |
| `map-group_gp01_hc_af_common_145.txt` | 正式共同样本分组文件 |
| `common_samples_145.txt` | 5 组学共同样本清单 |

`joint_selected_biomarkers_manifest.tsv` 是解释联合模型时必须查看的文件。它记录了每个 biomarker 来自哪一个单组学分支，便于判断联合模型是否被单一组学主导。

## 脚本来源

当前项目复用仓库级 `scripts/` 中的 `efs_valid_plus v0.2.2` 三步式入口；旧的 `lina_20251208` 路径保留为兼容实现位置：

```text
scripts/efs_valid_plus_step1.v0.2.2.R
scripts/efs_valid_plus_step2.v0.2.2.R
scripts/efs_valid_plus_step3.v0.2.2.R
```

三步含义：

| 阶段 | 作用 |
|---|---|
| Step1 | 数据读取、类型识别、预处理，输出 `preprocessed_data.RData` |
| Step2 | 特征选择、嵌套 CV、调参、9 种模型训练和测试集评估 |
| Step3 | 读取 Step2 结果，对已训练模型做 SHAP 解释 |

本项目新增的是 GP01.HC-AF 项目级封装：

| 脚本 | 作用 |
|---|---|
| `scripts/prepare_gp01_hc_af_inputs.R` | 生成 5 个单组学输入矩阵和共同样本分组文件 |
| `scripts/build_joint_biomarker_input.R` | 从 5 个单组学 Step2 入选特征生成 `Model 0.6` 联合输入 |
| `scripts/run_gp01_hc_af_analysis.sh` | 调度 prepare、single、joint 或 all |
| `scripts/run_gp01_shap_all.sh` | GP01 SHAP 兼容入口；实际调用仓库级 `scripts/run_mlr3_shap_all.sh` 批量运行 SHAP |
| `scripts/summarize_gp01_final_delivery.sh` | 调用仓库级 `scripts/summarize_mlr3_results.R`，统一生成模型性能、Step2 技术设置、SHAP 输出和总说明书 |

## 运行阶段划分

`run_gp01_hc_af_analysis.sh` 将正式流程拆成 4 个可单独运行的阶段：

| 阶段 | 含义 | 典型用途 |
|---|---|---|
| `prepare` | 生成 5 个单组学输入矩阵和共同样本分组 | 数据更新后先核对输入 |
| `single` | 运行 `Model 0.1-0.5` 单组学 Step1/Step2 | 单组学模型重跑 |
| `joint` | 从单组学 Step2 入选特征生成联合输入并运行 `Model 0.6` | 单组学结果已完成后补跑联合模型 |
| `all` | 从输入生成到单组学和联合模型一次跑完 | 完整复现 |

这种拆分继承了 `20260403_lilei` 的交付思路：输入、建模、解释和汇总可以分别检查，不需要每次为了补一个汇总或 SHAP 输出重跑全流程。

## 与 20260403_lilei 的关系

`20260403_lilei` 提供的是 v0.2.2 正式交付模式：入口 README、输入 manifest、Step1/Step2/Step3 分支目录、`01_model_performance_summary/`、`02_step2_technical_summary/` 和 `03_shap_output_summary/`。

当前项目保留同一交付结构，但不照搬 `20260403_lilei` 的任务编号、性能阈值或疗效预测策略；解释口径按 GP01.HC-AF 诊断问题处理。

| 项目 | `20260403_lilei` | `lina_20260515` |
|---|---|---|
| 主要任务 | 食管癌诊断、疗效预测、动态差值分析 | GP01.HC-AF 诊断建模 |
| 建模分支 | 多任务、多时间点 | 5 个单组学 + 1 个 biomarkers 联合模型 |
| 类别不平衡 | 部分疗效分支更明显 | `HC=46, AF=99`，约 2.15:1 |
| 结果整理 | 正式结果根目录 + 多个 summary 子目录 | 同样保留正式结果根目录和 summary 子目录 |
| 解读口径 | Balanced_Score 只作辅助，SHAP 用于解释 | 同样执行，不声明唯一最优模型 |
