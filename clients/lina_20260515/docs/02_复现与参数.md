# 02 复现与参数

## 正式入口

正式输入：

```text
lina_20260515/v0.2.2_inputs_gp01_hc_af
```

正式结果：

```text
lina_20260515/reproduce_gp01_hc_af_af_positive_fullrerun_20260526_1531
```

## 脚本角色

正式分析使用 `lina_20260515/scripts/` 下的项目级封装脚本，并复用 `lina_20251208` 的 v0.2.2 三步式建模脚本。脚本可分为两类：输入/分组整理脚本，以及建模结果统计脚本。通用建模结果统计逻辑维护在仓库级 `scripts/summarize_mlr3_results.R`，项目目录中只保留 GP01 最终交付包装入口。

### 输入和分组整理

| 脚本 | 用途 | 主要输出 |
|---|---|---|
| `prepare_gp01_hc_af_inputs.R` | 读取 5 套原始数据和 `map-group.txt`，生成 145 个共同样本输入矩阵 | `v0.2.2_inputs_gp01_hc_af/` |
| `build_joint_biomarker_input.R` | 从 5 个单组学 Step2 的 `selected_features` 抽取 biomarkers | `model_0.6_*joint_selected_biomarkers_145.tsv` |
| `run_gp01_hc_af_analysis.sh` | 调度 prepare、single、joint 或 all | `run_model_*_step1/step2` |
| `run_gp01_shap_all.sh` | GP01 兼容入口；实际调用仓库级 `scripts/run_mlr3_shap_all.sh` 自动扫描 Step2 分支并运行 SHAP；默认请求 `train,test,valid`，waterfall/force 图按 case/control 各 1 个样本输出 | 各 Step2 目录下的 `step3_shap_*` |

### 建模结果统计

| 脚本 | 用途 | 主要输出 |
|---|---|---|
| `scripts/summarize_mlr3_results.R` | 仓库级通用合并版结果统计脚本；自动扫描指定结果目录下所有 `run_model_*_step2`，用 `--tasks` 控制统计模块 | `01/02/03_*summary/` 和 `RESULT_INTERPRETATION_MANUAL.md` |
| `scripts/redraw_mlr3_efs_diagnostics.R` | 仓库级 EFS 诊断图重绘脚本；基于已有 `efs.RData` 只重绘 `step1.1` 到 `step1.5`，不改最终特征和模型结果 | `step1.*.efs*.pdf`、`efs_diagnostic_4learner.rds`、`efs_diagnostic_learners.tsv` |
| `summarize_gp01_final_delivery.sh` | GP01 shell 包装入口；建模和 SHAP 完成后运行一次，默认执行 `--tasks=all` | 全部正式汇总和总说明书 |

## 一键复现

从仓库根目录运行完整分析：

```bash
export CASE_GROUP=AF
export CONTROL_GROUP=HC
bash lina_20260515/scripts/run_gp01_hc_af_analysis.sh all \
  /Users/colinliu/Desktop/mlr3_docker_tutorial/lina_20260515/reproduce_gp01_hc_af_af_positive_manual
```

`run_gp01_hc_af_analysis.sh` 会根据 `CASE_GROUP` 和 `CONTROL_GROUP` 生成 `--gp AF-HC`。由于当前 Step2 使用 `positive = gp[1]`，诊断任务必须保证 `AF` 在 `--gp` 的第一位。脚本启动时会打印并检查：

```text
case=AF, control=HC, gp=AF-HC
AF=99, HC=46
```

只生成输入：

```bash
bash lina_20260515/scripts/run_gp01_hc_af_analysis.sh prepare
```

只运行 5 个单组学模型：

```bash
bash lina_20260515/scripts/run_gp01_hc_af_analysis.sh single \
  /Users/colinliu/Desktop/mlr3_docker_tutorial/lina_20260515/reproduce_gp01_hc_af_af_positive_manual
```

在单组学模型已完成后，只生成 biomarkers 联合输入并运行联合模型：

```bash
bash lina_20260515/scripts/run_gp01_hc_af_analysis.sh joint \
  /Users/colinliu/Desktop/mlr3_docker_tutorial/lina_20260515/reproduce_gp01_hc_af_af_positive_manual
```

## 正式参数

Step2 通用参数：

```text
--part 3/4
--split none
--n_features 15
--feature_fraction 0.5
--outer_cv 5
--inner_cv 3
--tune_evals 20
--tune_batch_size 5
--mlr3_log_level warn
--cores 1
--model_subset all
--feature_combine_method majority
--valid none
```

参数说明：

| 参数 | 含义 |
|---|---|
| `--split none` | 不提供独立外部验证集 |
| `--part 3/4` | 内部训练/测试划分使用 3/4 训练比例 |
| `--outer_cv 5` | 外层 5 折交叉验证 |
| `--inner_cv 3` | 内层 3 折交叉验证 |
| `--model_subset all` | 运行 9 种候选分类器 |
| `--feature_combine_method majority` | 多方法特征选择使用多数策略汇总 |
| `--feature_fraction 0.5` | 特征选择比例参数，不是分类阈值 |

9 种候选模型为 `glmnet`、`xgboost`、`rf`、`svm`、`naive_bayes`、`kknn`、`rpart`、`adaboost` 和 `nnet`。

注意 `step1.1.efs.performance.pdf` 到 `step1.5.efs.stability.pdf` 只展示 EFS 层的 learner，不等同于最终 9 种候选分类器。旧版 `lina_20251208/efs_valid.R` 的 EFS 层包含 `xgb`、`rf`、`svm` 和 `lgbm`；当前 v0.2.2 Step2 主脚本已恢复这 4 个 EFS-capable learner。正式 fullrerun 结果目录中的 6 个 Step2 分支均已在 `efs.RData` 的 `learners_efs` 中记录 `xgb,rf,svm,lgbm`。

本次 fullrerun 覆盖了 Step2 特征选择、模型训练、workbook、SHAP 和正式汇总。正式特征选择结果仍以后续 `method_used` 为准。本项目实际使用多数策略整合 Boruta、RFE、LASSO 和 XGBoost；口腔细菌属分支因 LASSO 未选出特征，实际为 Boruta、RFE 和 XGBoost。

## Step1 preset

Step1 preset 决定输入数据的基础识别和预处理口径：

| 模型 | 输入 | Step1 preset |
|---|---|---|
| `0.1` | 非靶代谢 | `metabolome_standard` |
| `0.2` | 口腔细菌属 | `microbiome_standard` |
| `0.3` | 口腔真菌属 | `microbiome_standard` |
| `0.4` | 肠道细菌属 | `microbiome_standard` |
| `0.5` | 肠道真菌属 | `microbiome_standard` |
| `0.6` | 5 组学 biomarkers 联合 | `mixed_omics` |

非靶代谢特征使用 `Metabolite_` 前缀，微生物属水平特征使用 `Genus_*` 前缀。联合模型使用 `mixed_omics`，因为输入来自多种组学来源。

## 分类阈值

当前 Step2 结果的 `analysis_flowchart.md` 中记录：

```text
threshold = youden
```

这表示分类阈值按 Youden 方法确定。当前正式结果不展示、也不采用“0.5 作为最优临界点”的结论。

各分支 `analysis_flowchart.md` 中还保留 `loocv_threshold = 100`。当前各分支样本量为 145，正式 Step2 使用外层 5 折、内层 3 折的嵌套 CV，不走 LOOCV 主线。

当前 fullrerun 的 Step2 workbook 已由主流程直接规范写出，不需要在正式汇总前运行阈值列修复脚本。旧结果如仍存在 `.x/.y/_thr` 展示问题，可显式运行修复脚本；该脚本会清理 `_thr` 后缀，并把 `Sensitivity` 和 `Specificity` 补写到 `混淆矩阵_Train/Test` 以及每个单模型 sheet 中：

```bash
Rscript scripts/repair_mlr3_threshold_columns.R \
  --root=/path/to/old_result_root
```

## 类别不平衡

正式共同样本：

```text
HC = 46
AF = 99
不平衡比例约 2.15:1
```

当前正式分析不默认启用 SMOTE：

```text
USE_SMOTE=FALSE
```

原因是当前不平衡程度属于轻中度，正式主线先保留原始分布，解读时同时看 AUC、MCC、Recall 和混淆矩阵。若后续需要敏感性分析，另开目录运行，不覆盖当前正式结果：

```bash
USE_SMOTE=TRUE bash lina_20260515/scripts/run_gp01_hc_af_analysis.sh all \
  lina_20260515/reproduce_gp01_hc_af_smote
```

## 汇总命令

建模和 SHAP 都完成后，推荐只运行一次合并版统计：

```bash
bash lina_20260515/scripts/summarize_gp01_final_delivery.sh \
  lina_20260515/reproduce_gp01_hc_af_af_positive_fullrerun_20260526_1531
```

或直接调用 R 脚本：

```bash
Rscript scripts/summarize_mlr3_results.R \
  --root=lina_20260515/reproduce_gp01_hc_af_af_positive_fullrerun_20260526_1531 \
  --branch-map=lina_20260515/v0.2.2_inputs_gp01_hc_af/input_summary.tsv \
  --title=GP01.HC-AF \
  --shap-dir-name=step3_shap_formal_case_control_force_20260526 \
  --tasks=all
```

`--tasks` 可设为 `performance`、`step2`、`shap`、`manual` 或用逗号组合，例如：

```bash
Rscript scripts/summarize_mlr3_results.R \
  --root=lina_20260515/reproduce_gp01_hc_af_af_positive_fullrerun_20260526_1531 \
  --branch-map=lina_20260515/v0.2.2_inputs_gp01_hc_af/input_summary.tsv \
  --title=GP01.HC-AF \
  --tasks=performance,step2
```

该脚本不写死模型目录名，而是访问指定 `--root` 下所有 `run_model_*_step2` 文件夹。`--branch-map` 可选；不提供时，分支名称从目录名推断。当前 GP01 项目使用 `v0.2.2_inputs_gp01_hc_af/input_summary.tsv` 作为 branch map，以保留中文组学名称。

批量运行当前正式 SHAP：

```bash
SHAP_LOCAL_DISPLAY_FEATURES=10 \
bash lina_20260515/scripts/run_gp01_shap_all.sh \
  lina_20260515/reproduce_gp01_hc_af_af_positive_fullrerun_20260526_1531
```

其他项目应优先直接调用仓库级通用脚本：

```bash
bash scripts/run_mlr3_shap_all.sh /path/to/result_root
```

当前正式 SHAP 使用脚本默认参数：

```text
SHAP_NSIM_TRAIN=20
SHAP_NSIM_TEST=12
SHAP_MAX_TRAIN_ROWS=60
SHAP_MAX_TEST_ROWS=60
SHAP_BACKGROUND_ROWS=80
SHAP_MIN_AUC=0.500001
SHAP_LOCAL_DISPLAY_FEATURES=10
```

`SHAP_LOCAL_DISPLAY_FEATURES=10` 只限制 waterfall/force 单样本局部解释图的展示特征数。当前正式 PDF 中，importance、beeswarm、waterfall 和 force 只对较长特征名自动换行，默认超过 28 个字符时优先在靠近中点的 `-` 后断为最多两行；短特征名不换行；force 保持较低的箭头形状单行正负累积贡献条，Positive 特征名按箭头起点对齐，Negative 特征名按箭头尾部右对齐。SHAP Excel 仍保留完整数值。
