# result_manifests

This directory stores lightweight result evidence copied from server runs.

It is not a replacement for full result directories. Full outputs remain on the server.

Use result manifests for:

- formal verification tables,
- SHAP status summaries,
- model file lists,
- expected output counts,
- audit reports.

Do not store:

- full PDF sets,
- full Excel result bundles,
- model RData objects,
- complete `run_model_*` trees.
